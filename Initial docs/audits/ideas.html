Kategoria pierwsza
Backend, Infrastruktura & Logika
01
Bezpieczeństwo
Prawdziwe RAG — Voyage AI zamiast Math.random()
Zamienić losowe embeddingi w knowledge.ts i memory.ts na wywołania Voyage AI lub text-embedding-3-small. Dodać ORDER BY embedding <=> $vector LIMIT 5 do knowledge search. Agenci zaczną naprawdę pamiętać wcześniejsze zadania i korzystać z bazy wiedzy — co jest przecież jednym z kluczowych selling points projektu.
~1 dzień
02
Architektura
Shared Types Package — koniec z podwójną definicją typów
Stworzyć packages/types/ z eksportami wszystkich encji DB (Agent, Task, Company, Goal…). Generować je ze schematu SQL przez kysely-codegen lub pg-to-ts. Frontend i backend przestają ręcznie synchronizować interfejsy — błędy typów wykrywane w czasie kompilacji, nie w runtime. Każda zmiana schematu automatycznie propaguje się do dashboardu przez TS.
~2 dni
03
Infrastruktura
node-pg-migrate — koniec z schema_v*.sql
Zastąpić własny runner przez node-pg-migrate z tabelą pgmigrations. Każda migracja to timestamp + up/down. Deployment produkcyjny staje się deterministyczny — nigdy więcej "nie wiem który stan ma baza".
~4h
04
Testy
Test Suite — Vitest + Testcontainers; zero excuses
Projekt ma 3200+ linii krytycznej logiki i zero testów. Start: safety.test.ts (8 przypadków: circular delegation, message flood, consecutive errors), policies.test.ts (5 typów polityk), heartbeat.test.ts (atomic checkout, budget enforcement). Narzędzia: Vitest dla szybkości, @testcontainers/postgresql dla izolowanej bazy na każdy test suite. CI wymagać 80% coverage na orchestrator/.
~3 dni
05
Architektura
Event Bus — Redis Streams zamiast poll + WebSocket push
Aktualny model: worker → bezpośrednio do WS hub przez importy. Problem przy skalowaniu: dwa procesy workera nie widzą nawzajem swoich eventów. Redis Streams jako event bus: każde zdarzenie (heartbeat, approval, task.done) trafia do streamu, WS hub subskrybuje, wszystkie instancje widzą wszystkie eventy. Umożliwia horizontal scaling worker bez zmian w dashboard.
~3 dni
06
Runtime
Anthropic Tool Use API — koniec z XML regex parserem
Zamiast parsować <thought> i <actions> regexem (łamie się przy długich odpowiedziach), użyć natywnego Anthropic Tool Use. Zdefiniować każdą akcję (complete_task, delegate, use_tool…) jako tool schema. Model zwraca strukturyzowane tool_use bloki — zero parsowania, 100% typesafe. Eliminuje całą klasę "agent nie działa bo model odpowiedział inaczej niż oczekiwano".
~2 dni
07
Obserwabilność
OpenTelemetry Tracing — pełna widoczność przez cały heartbeat cycle
Każdy heartbeat to 8+ kroków (budget check, task checkout, context build, LLM call, action processing…). Aktualnie traceback przez logi Pino. Z OpenTelemetry: każdy heartbeat to span, każdy krok to child span z timestamp, koszt LLM jako atrybut, błędy z stack trace. Eksport do Jaeger lub Tempo. Debugowanie "dlaczego agent nie wykonał zadania" spada z 30 minut do 30 sekund.
~2 dni
08
Bezpieczeństwo
Bash Sandbox — Docker exec z izolacją sieci i filesystem
Whitelist komend to dobry start, ale agent może nadal wykonać dozwoloną komendę (git, python) w destrukcyjny sposób. Docker exec z --network none --memory 128m --cpus 0.5 -v workspace:/w:rw --read-only daje prawdziwą izolację. Żaden kod agenta nie ma dostępu do sieci ani plików poza workspace.
~2 dni
09
Architektura
Template Marketplace — preset library z GitHub
Aktualny export pomija połowę danych firmy. Po naprawieniu: dodać publiczne repozytorium szablonów (content-agency.json, dev-shop.json, solo-founder.json). Endpoint GET /api/templates/presets pobiera listę z GitHub releases. Import jednym kliknięciem. Społeczność może PR-ować nowe szablony.
~3 dni
10
Biznes
Multi-Provider LLM Router — Portkey lub LiteLLM jako middleware
Aktualnie każdy runtime to osobna klasa z osobną logiką. LiteLLM proxy jako warstwa unifikuje wszystkich providerów pod jednym API (claude/, openai/, gemini/, groq/…). Zyski: automatyczny fallback gdy provider zwróci 429, load balancing między kluczami API, caching promptów dla powtarzalnych kontekstów, cost tracking w jednym miejscu. Można dodać nowy provider bez zmiany kodu orchestratora.
~2 dni
B
Kategoria druga
UI/UX & Doświadczenie Użytkownika
U1
Live Activity Feed — stream heartbeatów w czasie rzeczywistym z animacją
Sidebar lub dedykowana sekcja dashboardu pokazująca live scroll agentów przy pracy: każdy heartbeat pojawia się jako wiersz z nazwą agenta, myślą, kosztem i timestampem. Dane już są przez WebSocket — brakuje tylko komponentu React który podłącza useWebSocket() i renderuje przesuwający się log. Efekt: dashboard zaczyna "żyć", użytkownik widzi że coś się dzieje bez odświeżania.
Real-time
U2
OrgChart — interaktywna wizualizacja hierarchii agentów
Dane reports_to są w bazie od początku projektu; brakuje wyłącznie frontendu. D3.js force graph lub React Flow z węzłami agentów, kolorowanymi według statusu (idle/working/paused), połączonymi liniami hierarchii. Klik na agenta otwiera jego szczegóły. To jest wizualna tożsamość projektu — "firma złożona z agentów AI" nabiera sensu gdy widzi się org chart.
Flagship
U3
BudgetsPage — monitoring kosztów z prognozą end-of-month
API /agents/:id/budgets i /companies/:id/costs istnieje i zwraca dane. Brakuje strony która pokazuje: pasek wydane / limit USD per agent, wykres dzienny (Recharts sparkline), i prognoza do końca miesiąca oparta na średnim koszcie ostatnich 7 dni. Alert gdy agent osiągnie 80% limitu. To jedna z najważniejszych stron dla kogoś kto płaci za LLM API własną kartą.
Priorytet
U4
Mobile-first Approvals — szybkie zatwierdź/odrzuć z powiadomieniem push
Kluczowy flow: agent prosi o zatwierdzenie → użytkownik dostaje powiadomienie → zatwierdza na telefonie. Aktualnie ApprovalsPage jest responsywna ale brakuje Web Push Notifications (Service Worker + VAPID keys). Integracja z istniejącym NotificationService. Efekt: użytkownik może zarządzać firmą AI ze smartfona w 30 sekund gdy jest poza biurem.
Mobile
U5
Command Palette — ⌘K do wszystkiego w systemie
Użytkownicy zarządzający wieloma agentami i taskami potrzebują szybkiej nawigacji. ⌘K palette (np. cmdk library) z global search po agentach, taskach, celach, możliwością zatrudnienia agenta bez wchodzenia w formularz, i szybkimi akcjami (pause agent X, create task, view logs). Power-user feature który sprawia że dashboard czuje się jak profesjonalne narzędzie, nie kolejna aplikacja webowa.

WOW · 01
Agent Replay — cofnij czas i obejrzyj co robił agent
Audit log i heartbeats są w bazie. Zbudować time-travel debugger: slider przez historię firmy, rekonstrukcja stanu każdego agenta w dowolnym momencie, odtwarzanie decyzji krok po kroku z myślami agenta, kosztami i akcjami. Możliwość "fork" — cofnięcie decyzji i uruchomienie agenta z innym promptem od tego momentu. Nieistniejący feature w żadnym obecnym narzędziu do orkiestracji agentów.
Time-travel debugging
📡
WOW · 02
Agent-to-Agent Marketplace — wynajmij agenta innej firmy
Multi-tenant architektura jest już gotowa. Rozszerzyć o cross-company agent contracts: firma A może wystawić agenta (np. "Senior Developer, $0.05/task") na publicznym marketplace. Firma B zleca mu zadania przez standardowe API, płaci przez billing system, wyniki wracają jako wiadomości. Każdy deployment Autonomicznego Biura staje się węzłem w sieci. Pierwsza platforma gdzie firmy AI robią biznes między sobą.
Agent economy

WOW · 03
Naturalny język jako interfejs — "zatrudnij CTO który umie Rust"
Zamiast formularzy — conversational onboarding. Użytkownik pisze: "Chcę zbudować SaaS do zarządzania magazynem, zatrudnij mi zespół". System sam generuje org chart, tworzy agentów z odpowiednimi promptami, ustawia cele i polityki, i pyta tylko o zatwierdzenie. Pod spodem: Claude analizuje intent i tłumaczy na serię wywołań API. Czas od "chcę zacząć" do "firma działa" spada do 2 minut.
